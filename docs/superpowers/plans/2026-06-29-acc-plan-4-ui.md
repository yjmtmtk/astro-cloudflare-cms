# astro-cloudflare-cms ŌĆö Plan 4: UI (components, page routes, bundled CSS) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Steps use `- [ ]`.

**Goal:** Port the full admin + public UI from `nanocms` into the integration ŌĆö shadcn primitives, the React island components, the admin/news page routes injected via `injectRoute`, and a package-bundled Tailwind `styles.css` ŌĆö so the demo renders the complete working CMS (admin UI + `/news`) with NO Tailwind/shadcn setup in the host.

**Architecture:** Components and page routes compile in the HOST's Astro/Vite build, so all `@/...` imports become **relative**, and hardcoded URL prefixes (`/admin`, `/news`) become `config.adminBasePath`/`config.newsBasePath` read from the build-time-inlined `virtual:acc-config`. The package ships a compiled `styles.css` (Tailwind built over the package's own components ŌĆö validated in the POC); page routes import it. `/cms-media` and the cookie name stay literal (lib-internal). UI deps (shadcn radix primitives, tiptap, lucide, clsx/tailwind-merge/cva) become package runtime deps; `tailwindcss` is a dev dep for the CSS build.

**Tech Stack:** Plan 1ŌĆō3 + `clsx`, `tailwind-merge`, `class-variance-authority`, `lucide-react`, `@radix-ui/react-{slot,label,select,dialog,separator,tooltip}`, `@tiptap/{react,starter-kit,pm,extension-image}`; dev `tailwindcss`. React 19 (peer).
